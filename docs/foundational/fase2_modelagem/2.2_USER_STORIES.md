## Convenções deste documento

### Formato de cada User Story

> **[US-XXX]** Título curto
> **Ator:** Usuário | Fornecedor | Comprador | Admin | Sistema
> **Fase:** MVP | Validação | Monetização | Escala
> **Origem:** UC(s) de onde foi derivada
> **Rastreabilidade:** RFs e RNs relacionados

**Narrativa:**
Como [ator], quero [ação] para [benefício/valor de negócio].

**Critérios de aceite (CA):**
Condições verificáveis que definem "pronto". Escritos no formato Dado/Quando/Então (Given/When/Then) quando o cenário exigir clareza, ou como checklist objetivo quando for mais direto.

### Prioridade (MoSCoW por fase)

| Tag | Significado |
|---|---|
| **Must** | Obrigatório para a fase indicada |
| **Should** | Importante, mas entregável em iteração seguinte |
| **Could** | Desejável se sobrar tempo |
| **Won't (yet)** | Deliberadamente adiado |

### Estimativa de complexidade (Story Points — Fibonacci)

| SP | Referência |
|---|---|
| 1 | Tarefa trivial (ex: exibir campo somente leitura) |
| 2 | Tarefa simples com pouca lógica |
| 3 | Tarefa média (formulário com validações) |
| 5 | Tarefa complexa (integração com API externa) |
| 8 | Tarefa grande (fluxo completo com múltiplas regras) |
| 13 | Épico que precisa ser quebrado em sub-stories |

## CADASTRO UNIFICADO — MVP

### US-001: Cadastro unificado na plataforma (Nível 1)

**Ator:** Usuário (qualquer pessoa)
**Fase:** MVP | **Prioridade:** Must | **SP:** 5
**Origem:** UC-01
**Rastreabilidade:** RF-01.01, RF-01.05 | RN-01.03, RN-01.05, RN-01.13

**Narrativa:**
Como pessoa interessada, quero me cadastrar na plataforma de forma rápida e sem fricção para poder explorar produtos, fornecedores e decidir se quero comprar ou vender.

**Critérios de aceite:**

- CA-001.1: Formulário de cadastro exige: email, senha (mín. 8 chars, 1 maiúscula, 1 número), nome completo, telefone, cidade e estado. Sem CNPJ, sem dados de empresa, sem escolha de role.
- CA-001.2: Email já cadastrado é rejeitado com sugestão de login ou recuperação de senha.
- CA-001.3: Conta criada com status "pendente de confirmação" e sem role definido (Nível 1 — Usuário).
- CA-001.4: Email de ativação enviado. Link válido por 48h. Lembrete enviado automaticamente se não confirmado. Conta removida após 7 dias sem confirmação.
- CA-001.5: Após confirmação, conta ativada como Nível 1 (Usuário). Pode navegar, buscar, ver catálogo e perfis públicos.
- CA-001.6: Usuário NÃO pode enviar inquiries (precisa ativação de buyer, UC-12) nem listar produtos (precisa upgrade para supplier, UC-31).
- CA-001.7: CPF nunca coletado (RN-01.13 — dado sensível, sem valor de negócio para marketplace B2B).
- CA-001.8: Senha não atende requisitos mínimos exibe critérios claros ao lado do campo.

## FORNECEDOR — MVP

### US-002: Cadastro provisório quando API da Receita estiver indisponível

**Ator:** Fornecedor
**Fase:** MVP | **Prioridade:** Must | **SP:** 3
**Origem:** UC-01 (EX-01.2), UC-24
**Rastreabilidade:** RF-01.02, RF-01.03 | RN-01.01, RN-01.02

**Narrativa:**
Como fornecedor, quero que meu cadastro não seja bloqueado quando a API da Receita Federal estiver fora do ar para que eu não perca o interesse e abandone a plataforma.

**Critérios de aceite:**

- CA-002.1: Quando a API retorna erro ou timeout, sistema cria conta com flag `cnpj_pending_validation`.
- CA-002.2: Sistema agenda revalidação automática em 24h (retry a cada 24h, até 72h).
- CA-002.3: Se após 3 tentativas (72h) a API permanecer indisponível, admin é notificado.
- CA-002.4: Fornecedor com CNPJ pendente pode acessar o painel, mas o badge "CNPJ Verificado" não é exibido até a validação concluir.

### US-003: Editar perfil da empresa

**Ator:** Fornecedor
**Fase:** MVP | **Prioridade:** Must | **SP:** 5
**Origem:** UC-02
**Rastreabilidade:** RF-02.01, RF-02.02, RF-02.03, RF-02.04, RF-02.05 | RN-02.01

**Narrativa:**
Como fornecedor, quero editar o perfil da minha empresa (logo, descrição, endereço, categorias, fotos) para que compradores encontrem informações completas e minha posição na busca melhore.

**Critérios de aceite:**

- CA-003.1: Campos editáveis: logo, descrição (até 2.000 chars), endereço completo, horário de funcionamento, site, redes sociais, ano de fundação, faixa de funcionários.
- CA-003.2: Fornecedor seleciona até 5 categorias/setores na árvore hierárquica.
- CA-003.3: Upload de até 10 fotos da empresa (máx. 5MB cada, formatos JPG, PNG, WebP).
- CA-003.4: Imagens >5MB ou em formato inválido são rejeitadas individualmente com mensagem clara.
- CA-003.5: Salvamento parcial aceito (fornecedor não precisa preencher tudo de uma vez).
- CA-003.6: Barra de completude recalculada em tempo real conforme fórmula RN-02.01: Logo (10%) + descrição ≥100 chars (15%) + endereço (10%) + telefone (10%) + 1+ categoria (10%) + 3+ produtos (20%) + 1+ foto/produto (15%) + horário (5%) + ano fundação (5%).
- CA-003.7: Após salvar, sistema exibe quais campos faltam para chegar a 100%.
- CA-003.8: Posição no ranking de busca recalculada após salvar.
- CA-003.9: Descrição com conteúdo suspeito (spam, links externos) é salva mas marcada para moderação.

### US-004: Cadastrar produto

**Ator:** Fornecedor
**Fase:** MVP | **Prioridade:** Must | **SP:** 5
**Origem:** UC-03
**Rastreabilidade:** RF-03.01, RF-03.02, RF-03.04, RF-03.05, RF-03.06 | RN-02.02, RN-02.03, RN-02.04, RN-02.05, RN-02.06, RN-02.07

**Narrativa:**
Como fornecedor, quero cadastrar meus produtos com descrição, fotos e categoria para que compradores encontrem o que procuram e me enviem cotações.

**Critérios de aceite:**

- CA-004.1: Formulário com campos: nome (obrigatório), descrição até 1.000 chars (obrigatório), categoria + subcategoria (obrigatório), fotos (até 5, máx. 5MB), unidade de medida, quantidade mínima de pedido, faixa de preço (opcional).
- CA-004.2: Campos obrigatórios não preenchidos são destacados com mensagem de erro.
- CA-004.3: Sistema gera tags/palavras-chave automaticamente a partir do nome e descrição.
- CA-004.4: Sistema gera slug SEO-friendly para URL no padrão `/produto/[slug]-[cidade]`.
- CA-004.5: Produto sem foto é aceito, mas sistema exibe aviso "Produtos com fotos recebem 3x mais inquiries".
- CA-004.6: Produto sem preço exibe "Preço sob consulta" na página pública.
- CA-004.7: Produto duplicado (mesmo nome + mesma categoria do mesmo fornecedor) gera aviso pedindo confirmação.
- CA-004.8: Sitemap XML atualizado após publicação.
- CA-004.9: Completude do perfil recalculada (critério de 3+ produtos vale 20%).
- CA-004.10: Listagem de produtos é ilimitada e gratuita para todos os fornecedores (RN-02.02).

**Ator:** Fornecedor
**Fase:** Monetização | **Prioridade:** Must | **SP:** 8
